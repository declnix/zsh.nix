# Agent Instructions

## Git Commit Convention

Use Conventional Commits for commit messages:

```text
type(scope): subject
```

- Prefer `feat`, `fix`, `docs`, `test`, `refactor`, and `chore`.
- Use an optional scope when it clarifies the affected area.
- Keep the subject lowercase, imperative, concise, and without a trailing period.
- Good scopes for this repo include `wrapper`, `plugins`, `checks`, `treefmt`, `flake`, and `docs`.

Examples:

```text
feat(plugins): add zsh plugin module
fix(wrapper): preserve plugin load order
docs: describe project structure
```

## Project Structure

- `flake.nix` is generated by `flake-file`; update source modules and run `nix run .#write-flake` when regeneration is needed.
- `modules/` is loaded through `import-tree`; new module files must be tracked by git before flake checks can see them.
- `modules/+wrapper/` contains the zsh wrapper app, library, options, renderer, and plugin modules.
- `modules/+wrapper/modules/plugins/` contains grouped plugin modules such as completion, editing, history, integrations, and syntax highlighting.
- `modules/checks/` contains smoke, opt, and startup performance checks.
- `modules/git-hooks.nix` and `modules/treefmt.nix` configure pre-commit and formatting behavior.
- `.agents/skills/` contains repo-local agent skills, including guidance for adding zsh plugin modules.
